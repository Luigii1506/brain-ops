# MVP Scope

## MVP definition

The MVP is the first usable version of a local Jarvis-style system.

It must already support:
- Obsidian knowledge operations
- deterministic local execution
- local AI readiness
- structured data readiness for life tracking

## Must have

- Python CLI
- explicit local config
- safe vault file operations
- note templates
- note creation and project scaffolding
- inbox processing
- frontmatter normalization
- note improvement
- grounded note research
- internal link suggestion and application
- note promotion
- weekly review and audit reports
- SQLite initialization for life-ops storage
- configuration ready for OpenClaw + Ollama

## Should have

- dry-run support on most write paths
- reports for bulk changes
- clear folder and note ontology
- source-to-knowledge workflow
- single-note enrichment pipeline

## Next after MVP

- nutrition logging
- workout logging
- expense logging
- daily summaries
- domain-aware command routing from OpenClaw

## Not required yet

- web UI
- public API
- SaaS auth
- multi-user permissions
- mobile app
- public deployment
- plugin marketplace
