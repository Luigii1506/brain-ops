# Project Overview

## What this project is

This repository is evolving from a personal life-ops assistant into a reusable operational core.

It is no longer just a system for tasks, diet, expenses, and fitness.

It is becoming a modular platform for:

- personal operations
- knowledge capture
- monitoring and alerts
- automation workflows
- structured data capture
- reusable APIs
- future domain modules

## Core idea

The project should become a "mother project" or reusable operational core that can support:

- life ops
- knowledge ops
- web monitoring
- scraping jobs
- alerts
- daily/weekly reviews
- future AML/research/market intelligence modules

## Main principles

- Conversation is an interface, not the core.
- OpenClaw is an orchestrator/surface, not the business brain.
- brain-ops is the real operational brain.
- SQLite stores structured operational data.
- Obsidian stores durable knowledge, summaries, documentation, and linked notes.
- Ollama is used as an auxiliary intelligence layer for parsing, extraction, and semantic tasks.
- The system must be reusable beyond personal use.

## Non-goals

- Do not redesign everything from scratch unless necessary.
- Do not center the architecture around Telegram or OpenClaw.
- Do not make the project depend on a single UI or interface.
- Do not move durable structured operational state into Obsidian.
