# Recommended Vault Structure

```text
00 - Inbox/
01 - Projects/
02 - Areas/
03 - Knowledge/
04 - Sources/
05 - MOCs/
06 - Systems/
07 - Archives/
Daily/
Templates/
Assets/
```

## Folder intent

### 00 - Inbox
Raw capture. Temporary. Must be processed regularly.

### 01 - Projects
Anything with a concrete outcome, active work, or bounded initiative.

Examples:
- AML Screening
- Ohara TCG
- OpenClaw
- brain-ops
- OR-Tools learning

### 02 - Areas
Long-term ongoing responsibilities:
- Health
- Career
- Finance
- Learning
- Reading

### 03 - Knowledge
Permanent notes, concepts, mental models, technical ideas, and reusable understanding.

### 04 - Sources
Articles, papers, docs, books, and material that has been ingested or summarized.

### 05 - MOCs
Maps of content. Navigation hubs.

### 06 - Systems
SOPs, runbooks, prompts, standards, procedures, and workflows.

### 07 - Archives
Inactive, completed, deprecated, or preserved material.

## Note types

Use frontmatter `type` values such as:
- project
- source
- permanent_note
- moc
- sop
- runbook
- study_note
- script_doc
- decision
- idea
- debugging_note
- prompt

## Required mindset

The vault is not a dumping ground.  
Every important note should become one of:
- reusable knowledge,
- operational documentation,
- structured source material,
- navigable project context.
